Service Communication Pattern

All services communicate through the API Gateway using REST APIs. Asynchronous tasks are handled via Celery with Redis as the message broker. Real-time updates use WebSocket connections through the API Gateway.

  • Synchronous: REST API calls for immediate responses
  • Asynchronous: Celery tasks for background processing
  • Real-time: WebSocket for live updates (leaderboards, notifications)
  • Event-driven: Webhooks for external integrations
